# Koffy v0.2.0 Release Notes

Koffy v0.2.0 adds Alipay web payments and rounds out the production self-hosting story after the initial v0.1.0 open-source release.

## Highlights

- Added Alipay recharge support for desktop website payment and mobile H5 payment.
- Added Alipay async notify handling with signature verification, amount checks, idempotent payment events, and wallet crediting.
- Added local test-mode hooks for Alipay notifications.
- Added user-center payment method selection with WeChat Pay and Alipay brand icons.
- Added independently uploadable and persistent favicons for the user center and admin console.
- Expanded the production Compose example so a blank Docker host can run the full stack.
- Updated LiteLLM defaults to the stable `latest` image tag.

## Fixes

- Localized Alipay recharge wallet ledger labels in the user center.
- Parsed Alipay callback timestamps as `Asia/Shanghai` when Alipay sends timezone-less payment times.
- Translated common Alipay gateway failures into clearer Koffy error codes and messages.

## Upgrade Notes

Existing v0.1.0 databases must run:

```sql
SOURCE migrations/003_add_alipay.sql;
```

Fresh installations do not need this extra step because `migrations/001_init.sql` already includes the updated provider enum.

Before enabling Alipay in production, configure `ALIPAY_*` variables, mount the required private key and public key or certificate files, and ensure the notify URL is public HTTPS:

```text
https://<your-domain>/api/v1/payments/alipay/notify
```
